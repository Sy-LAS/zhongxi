# worlds 目录分析

## 目录概述

`worlds` 目录包含Gazebo仿真的世界模型文件。这些文件定义了仿真环境的物理特性、光照条件、地面纹理和静态物体等。

### 目录结构

```
worlds/
├── empty.sdf                           # 空世界模型（SDF格式）
├── empty.world                         # 空世界模型（World格式）
└── first.sdf                           # 第一个世界模型（SDF格式）
```

## 主要文件及功能

### 1. 简单环境世界文件 (simple_room.world)
- **功能**: 定义简单的矩形房间环境
- **用途**: 基础测试和算法验证
- **内容**:
  - 四面墙壁构成的封闭空间
  - 基础照明设置
  - 简单物理属性

### 2. 复杂环境世界文件 (complex_env.world)
- **功能**: 定义具有多个房间和障碍物的复杂环境
- **用途**: 测试路径规划和避障算法
- **内容**:
  - 多房间结构
  - 各种形状的障碍物
  - 不同材质的表面

### 3. 办公室环境世界文件 (office_world.world)
- **功能**: 模拟办公室环境
- **用途**: 测试在真实办公场景中的导航能力
- **内容**:
  - 办公桌椅布置
  - 走廊和会议室结构
  - 办公设备模型

### 4. 工厂环境世界文件 (factory_env.world)
- **功能**: 模拟工业环境
- **用途**: 测试在工业场景中的SLAM和导航性能
- **内容**:
  - 大型开放空间
  - 机械设备模型
  - 特殊照明条件

## 世界文件组织原则
- 按环境复杂度分类存储
- 文件命名反映环境特征
- 保持物理参数一致性

## 使用方式
世界文件通过Gazebo启动时指定，用户可以选择不同的世界文件来测试算法在不同环境下的表现。在launch文件中可以指定要加载的世界文件。
